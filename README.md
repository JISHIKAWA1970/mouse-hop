# Mouse Hop

Mouse Hop は、マルチディスプレイ環境でマウスカーソルを次のディスプレイ中央へ移動する Windows 向けの軽量常駐アプリです。

## v0.6 の機能

- 起動するとタスクトレイに常駐します。
- 初期ホットキーは `Ctrl + Alt + F12` です。
- タスクトレイの `設定` からホットキー、移動方式、ディスプレイ順、標準フォルダへの配置、Windows 起動時の自動起動を変更できます。
- 設定したホットキーを押すたびに、現在カーソルがあるディスプレイを判定し、選択した移動方式とディスプレイ順に従って次のディスプレイ中央へ移動します。
- `ループ` では最後のディスプレイの次に最初のディスプレイへ戻り、`往復` では端のディスプレイで移動方向を反転します。
- 現在カーソルがあるディスプレイを判定できない場合は、内部の巡回カウンターで次のディスプレイへ移動します。
- タスクトレイの右クリックメニューから以下を実行できます。
  - `次のディスプレイへ移動`
  - `設定`
  - `一時停止` / `再開`
  - `終了`
- 終了時にグローバルホットキーを解除します。
- ホットキー登録に失敗してもアプリは終了せず、タスクトレイに常駐し続けます。
- Mouse Hop 専用の簡易トレイアイコンをコード内で生成して表示します。生成に失敗した場合は標準アイコンにフォールバックします。
- Windows 起動時の自動起動を設定画面で ON / OFF できます。
- 常用時の標準配置先は `%LOCALAPPDATA%\Programs\MouseHop\MouseHop.exe` です。

## 対応環境

- Windows 10
- Windows 11
- .NET 8 SDK

## ビルド方法

```powershell
dotnet build
```

Release ビルドを作成する場合は以下を実行します。

```powershell
dotnet build -c Release
```

## 配布用 exe の publish 方法

.NET 8 Runtime が入っている Windows x64 PC で軽量に使う場合は、self-contained ではない単一ファイル publish を基本にします。

```powershell
dotnet publish src/MouseHop/MouseHop.csproj -c Release -r win-x64 --self-contained false -p:PublishSingleFile=true -o publish
```

publish 後の実行ファイルは以下に作成されます。

```text
publish/MouseHop.exe
```

まずは `publish/MouseHop.exe` から起動できます。ただし Downloads や `publish/` フォルダから直接常用すると、フォルダ整理やファイル移動で自動起動が壊れやすくなります。常用する場合は、設定画面から標準フォルダへ配置してください。`publish/` と `publish-self-contained/` フォルダは Git 管理対象外です。

.NET 8 Runtime が入っていない PC 向けには Runtime 同梱版も publish できます。ファイルサイズは大きくなりますが、配布先で Runtime 不足によるトラブルは減ります。

```powershell
dotnet publish src/MouseHop/MouseHop.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o publish-self-contained
```

## 実行方法

```powershell
dotnet run --project src/MouseHop/MouseHop.csproj
```

起動後、通知領域に `Mouse Hop` のアイコンが表示されます。

## 使い方

1. Mouse Hop を起動します。
2. 初期設定では `Ctrl + Alt + F12` を押します。
3. カーソルが現在のディスプレイの次のディスプレイ中央へ移動します。
4. もう一度同じホットキーを押すと、さらに次のディスプレイ中央へ移動します。
5. タスクトレイアイコンを右クリックし、`終了` を選ぶとアプリを終了します。

## マウス追加ボタンで使う場合

Mouse Hop は、ユーザーがキーボードで複雑なショートカットを押すためのアプリではありません。マウス設定アプリ側で追加ボタンにショートカットを割り当て、そのショートカットを Mouse Hop が受け取る設計です。

初期設定では、マウス設定アプリ側で追加ボタンに `Ctrl + Alt + F12` を割り当てると、Mouse Hop をマウスボタン 1 つで実行できます。

Logi Options+ などのマウス設定アプリで別のキー組み合わせを割り当てたい場合は、Mouse Hop のタスクトレイメニューから `設定` を開き、同じキー組み合わせをホットキーとして登録してください。`Ctrl` / `Alt` / `Shift` / `Win` などの修飾キー付きショートカットに対応しています。`F13` のような単独キーも、入力できる環境では登録できます。

## 設定

1. タスクトレイアイコンを右クリックします。
2. `設定` を選びます。
3. 現在のホットキー、移動方式、ディスプレイ順を確認します。
4. ホットキーを変える場合は `変更` を押します。
5. 次に押したキー組み合わせが新しいホットキーとして保存されます。
6. 移動方式を変える場合は、`移動方式` から `ループ` または `往復` を選びます。
7. ディスプレイ順を変える場合は、`ディスプレイ順` の一覧でディスプレイを選び、`上へ` / `下へ` で並べ替えます。
8. 常用する場合は、`標準フォルダへ配置` を押して `%LOCALAPPDATA%\Programs\MouseHop\MouseHop.exe` へコピーし、標準配置先から再起動します。
9. Windows 起動時に自動起動したい場合は、標準配置後に `Windows 起動時に Mouse Hop を起動する` を ON にします。

設定は AppData 配下の `MouseHop/settings.json` に保存され、次回起動時にも反映されます。既存の settings.json に移動方式が存在しない場合は、初期値として `ループ` を使用します。ディスプレイ順が存在しない場合は、自動順序を使用します。ホットキー変更時は古いホットキーを解除してから新しいホットキーを登録します。自動起動設定は settings.json にも保存しますが、実際の表示は Windows の Run キー状態を優先します。

## 移動方式

ディスプレイ順が未設定の場合、接続中のディスプレイは画面上の配置に基づいて上から下、同じ高さでは左から右の順に並びます。メインディスプレイが左端または上端にない構成や、負の座標を含む配置にも対応します。

### ループ

現在までの挙動と同じ方式です。最後のディスプレイまで移動したら、次は最初のディスプレイへ戻ります。

例: `1 → 2 → 3 → 1 → 2 → 3`

### 往復

端のディスプレイまで移動したら、前回の移動方向を反転して戻ります。

例: `1 → 2 → 3 → 2 → 1 → 2 → 3`

2 画面の場合は、ループも往復も実質的に `1 → 2 → 1 → 2` です。

## ディスプレイ順

設定画面の `ディスプレイ順` では、接続中のディスプレイを手動で並べ替えできます。各ディスプレイは `Screen.DeviceName`、解像度、座標、Primary 情報で表示されるため、Windows の画面配置やディスプレイ番号だけでは作業感覚に合わない環境でも、使いたい順序を指定できます。

手動指定した順序は、`ループ` と `往復` の両方で使われます。たとえば手動順序を `3 → 2 → 1` にした場合、`ループ` は `3 → 2 → 1 → 3`、`往復` は `3 → 2 → 1 → 2 → 3` のように移動します。

接続ディスプレイ構成が変わった場合、現在存在するディスプレイを保存済み順序に従って優先し、存在しないディスプレイは無視します。新しく増えたディスプレイは、自動順序に従って末尾に追加します。ディスプレイ順が未設定の場合は、従来の自動順序を使用します。

## 標準配置先

Mouse Hop の常用時の標準配置先は以下です。

```text
%LOCALAPPDATA%\Programs\MouseHop\MouseHop.exe
```

例:

```text
C:\Users\<UserName>\AppData\Local\Programs\MouseHop\MouseHop.exe
```

配布された `MouseHop.exe` は Downloads や任意フォルダから最初に起動できますが、その場所をそのまま常用・自動起動登録することは推奨しません。設定画面に現在の場所と標準の場所が表示され、`標準フォルダへ配置` を押すと標準配置先へコピーして、標準配置先の `MouseHop.exe` を起動します。起動に成功した場合、元のプロセスは終了します。

`dotnet run` や DLL 経由の実行中は自己配置の対象外です。その場合は publish 済みの `MouseHop.exe` から起動して配置してください。

## Windows 起動時の自動起動

設定画面の `Windows 起動時に Mouse Hop を起動する` を ON にすると、標準配置先の `MouseHop.exe` を CurrentUser の Run キーに登録します。標準配置先に `MouseHop.exe` が存在しない場合は、先に標準フォルダへ配置する必要があります。

```text
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
値名: MouseHop
```

登録値は、標準配置先の実行ファイルパスをダブルクォート付きで保存します。OFF にすると同じ `MouseHop` の値を削除します。

Run キーに Downloads、`publish/`、`bin/`、一時フォルダなど標準配置先以外の古い MouseHop パスが入っている場合、Mouse Hop は自動起動 OFF として扱います。settings.json と Run キー状態が食い違う場合は、Run キー状態を優先します。

## 専用トレイアイコン

v0.4 では Mouse Hop 専用の小さな簡易アイコンをコード内で生成します。通知領域では、マウスカーソルと画面移動をイメージしたアイコンが表示されます。アイコン生成に失敗した場合もアプリはクラッシュせず、従来どおり標準アプリケーションアイコンを使用します。

## 既知の制限

- インストーラーはありません。
- コード署名はまだ行っていません。未署名アプリのため、Windows SmartScreen などの警告が出る場合があります。
- Microsoft Store 配布にはまだ対応していません。
- アップデーターはまだありません。
- 移動時のオーバーレイ表示はありません。
- 設定したホットキーが他のアプリで使用中の場合、ホットキー登録に失敗することがあります。その場合もタスクトレイには常駐し、右クリックメニューの `次のディスプレイへ移動` は使用できます。
